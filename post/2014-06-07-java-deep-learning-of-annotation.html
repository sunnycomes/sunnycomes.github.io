Java的@符号

2014-06-07 14:23:59 +080

Java中, 经常碰到"@"这个符号, 用于表示Java中的annotation, 我们通常用于修饰java.lang.Class, java.lang.reflect.Method, java.lang.reflect.Field以及java.lang.annotation.Annotation, 很多时候, 我们会将Java的反射机制跟annotation一起使用。

编程过程中, 常见的有@Override, @Deprecated, @SuppressWarnings, 或者更见多识广一些, 还有@Retention, @Target, @Documented, @Inherieted, 但是, 大多数情况下, 很少会有人去过问为什么这么去写, 只知道这是一种理所当然的写法. 在这里推荐一篇非常详细的关于Java注解的博文Java注解annotation用法和自定义注解处理器。

如果你使用过一些Java框架, 如Spring, 编程过程中会用到很多annotation, @Autowired, @Configuration, @Component, @Repository, @Service这些都经常用到. 这里我们探讨下, Spring是如何实现IoC, 其实原理很浅显, annotation + reflection, 我们在使用Spring的时候, 通常会指定bean所在的包名, 如下:

<context:component-scan base-package="com.iovi.ioc.controller"/>

这样做是为了告诉Spring容器, bean的路径, 通过遍历路径, 就能找到所有bean的具体位置, 通过反射机制, 做进一步的操作, 这个时候, annotation的作用就体现了, Spring容器通过获得bean的annotation, 然后遍历, 确定bean的类型, 然后实例化, 并且添加到BeanFactory. 一旦某些类中有@Autowire标记, 就从BeanFactory中查找, 然后注入.

Annotation type declarations are similar to normal interface declarations.
An at-sign (@) precedes the interface keyword. Each method declaration defines an element of the annotation type. Method declarations must not have any parameters or a throws clause.
Return types are restricted to primitives, String, Class, enums, annotations, and arrays of the preceding types. Methods can have default values.

再啰嗦一句, 在Annotation里面, 有一个接口很少抛头露面, 但有重要到不得不提的地步, 它就是java.lang.annotation.Annotation, 默认的, Override, Deprecated等annotation类型都继承于它, 跟java.lang.Object类似。更多详细资料见官网Interface Annotation. 值得注意的是, 你要是自己写个接口Intf, 实现java.lang.annotation.Annotation, 并不能使Intf成为java.lang.annotation.Annotation类型。
